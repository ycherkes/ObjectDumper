# Summary of Changes — February 15, 2026

## Settings UI Implementation

### 1. ObjectDumperSettings.kt — Full Rewrite

Replaced the minimal settings class (6 settings with wrong defaults) with a complete implementation matching the Visual Studio extension's `ObjectDumperOptionPage.cs`.

- **70+ settings** organized by format: Common, C#, VB, JSON, XML, YAML
- All defaults match the VS extension exactly (e.g., `commonMaxDepth=25`, `commonOperationTimeoutSeconds=10`, `csharpMaxCollectionSize=Int.MAX_VALUE`)
- **12 enums** with `jsonValue` properties for PascalCase JSON serialization:
  - `DateTimeInstantiation`, `DateKind`, `CollectionLayout`, `NamingStrategy`, `NamingConvention`, `TypeNameHandling`, `DateTimeZoneHandling`, `SortDirection`
  - `BindingFlagsModifiers`, `BindingFlagsInstanceOrStatic` — decomposed from .NET `BindingFlags` for UI-friendly dropdowns
- `computeBindingFlags()` method converts dropdown selections to .NET integer bitmask (Public=16, NonPublic=32, Instance=4, Static=8)

### 2. ObjectDumperConfigurable.kt — Full Rewrite

Replaced the flat single-panel form with a tabbed settings UI.

- **6 tabs**: Common, C#, Visual Basic, JSON, XML, YAML
- Each tab contains all relevant settings with proper form controls (checkboxes, dropdowns, text fields)
- BindingFlags presented as two dropdowns (Modifiers + Instance/Static), matching VS Code's approach
- `GetFieldsBindingFlags` uses an "Enable Get Fields" checkbox — unchecked sends `null` to the serializer
- `SortDirection` uses a nullable combobox with `(None)` display for the null value
- Full `isModified()`, `apply()`, `reset()` cycle implemented for all settings

### 3. DebuggerInteractionService.kt — buildOptionsJson() Updated

Replaced hardcoded JSON strings with dynamic JSON built from settings.

- All 5 format methods (`buildCSharpOptionsJson`, `buildVisualBasicOptionsJson`, `buildJsonOptionsJson`, `buildXmlOptionsJson`, `buildYamlOptionsJson`) now read every value from `ObjectDumperSettings`
- JSON properties in alphabetical order with PascalCase names, matching the VS extension's Newtonsoft.Json `StringEnumConverter` output
- `escapeJsonString()` helper for safe serialization of indent string and numeric format
- BindingFlags computed as integer bitmask via `computeBindingFlags()`
- SortDirection serialized as `"Ascending"`/`"Descending"` string or `null`

## DLL Copy Fix (SerializationExecutor.kt)

Fixed `copyDllToTempDirectory()` which previously had three problems:
1. Always copied to the same flat path `<temp>/YellowFlavor.Serialization.dll` — different framework DLLs would overwrite each other
2. Always overwrote on every debug session, even if unchanged
3. No deduplication

**New behavior:**
- Computes SHA-256 hash of the source DLL file
- Creates `<temp>/ObjectDumper/<hash>/YellowFlavor.Serialization.dll`
- Each distinct DLL version (per framework target) gets its own unique directory — no collisions
- If the file already exists at the hash-based path with matching size, the copy is skipped
