## DLL Copy Fix (SerializationExecutor.kt)

Fixed `copyDllToTempDirectory()` which previously had three problems:
1. Always copied to the same flat path `<temp>/YellowFlavor.Serialization.dll` â€” different framework DLLs would overwrite each other
2. Always overwrote on every debug session, even if unchanged
3. No deduplication

**New behavior:**
- Computes SHA-256 hash of the source DLL file
- Creates `<temp>/ObjectDumper/<hash>/YellowFlavor.Serialization.dll`
- Each distinct DLL version (per framework target) gets its own unique directory â€” no collisions
- If the file already exists at the hash-based path with matching size, the copy is skipped

## Keyboard Shortcuts (plugin.xml)

Added two-keystroke chord shortcuts to all 5 dump actions, supporting both Windows/Linux and macOS keymaps.

**Chord prefix:** `Ctrl+Shift+D` (Windows/Linux) / `Cmd+Shift+D` (macOS)

| Shortcut | Action |
|---|---|
| `Ctrl+Shift+D, D` | Dump as C# |
| `Ctrl+Shift+D, J` | Dump as JSON |
| `Ctrl+Shift+D, X` | Dump as XML |
| `Ctrl+Shift+D, V` | Dump as Visual Basic |
| `Ctrl+Shift+D, Y` | Dump as YAML |

- `$default` keymap entries use `ctrl shift D` (covers Windows/Linux)
- `Mac OS X 10.5+` keymap entries use `meta shift D` (covers macOS)
- `Ctrl+Shift+D` was chosen over `Ctrl+E` (used by VS extension) because `Ctrl+E` is bound to "Recent Files" in Rider's default keymap
